# Skill 使用指南

## 🎯 第一次使用的正确姿势

### 步骤 1：打开项目
在 Claude Code 中打开本项目目录。Agent 会自动对你说：

> 你好！我是你的经济学实证论文写作助手 📝
> 
> 我可以帮你：
> • 🆕 创建新论文项目
> • 📋 列出/切换已有论文
> • 📊 引导你完成选题 → 文献 → 数据 → 实证 → 论文的全流程
> • ⚙️ 自动执行 Stata 回归、生成 LaTeX 表格
> 
> 当前项目：demo-paper
> 
> 你想做什么？

### 步骤 2：说一句话开始

**不要敲命令**，直接说你想做什么。以下是一些示例：

| 你想说的话 | 效果 |
|-----------|------|
| `创建一篇关于最低工资与企业就业的论文` | 创建新项目 min-wage-employment，进入选题引导 |
| `看看 demo-paper 做到哪了` | 显示 demo-paper 的当前阶段和历史 |
| `列出我所有的论文` | 展示所有项目的进度表格 |
| `我想继续写数字经济那篇论文` | 切换到 demo-paper 项目 |
| `这个阶段完成了，下一步` | 推进到下一阶段 |

## ⚠️ 常见问题

### Q：我需要记住什么命令吗？

**不需要**。你想到什么就说什么，Agent 会自动理解你的意图并执行对应的操作。

如果 Agent 没听懂，它会用一句话追问你，而不是让你查文档。

### Q：我可以同时写多篇论文吗？

**可以**。每个项目独立隔离，数据、分析、状态互不干扰。

说一句"列出所有论文"就能看到所有项目的进度。

### Q：Stata 命令需要我自己敲吗？

**不需要**。Agent 会自动生成 .do 文件，通过 stata-mcp 在后台执行，然后把结果总结给你。

你只需要告诉它："跑一下基准回归"。

### Q：我想看看 demo-paper 的完整示例？

```
你：切换到 demo-paper 项目，然后看看这个项目的完整成果
```

## 🧠 Agent 的工作方式（你不需要知道，但可以了解）

当你说一句话时，Agent 在后台做了这些事：

1. **理解意图**：从你的自然语言中识别出你想做什么
2. **执行命令**：在后台调用 `scripts/scripts/pipeline.py <对应的操作>`
3. **转化输出**：把命令的原始输出转化为友好的自然语言
4. **引导下一步**：根据当前阶段，告诉你接下来可以做什么

所有技术细节对你透明。你只需要专注于研究本身。
